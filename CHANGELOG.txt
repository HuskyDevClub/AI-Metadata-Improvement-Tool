## Version 1.1.0 - March 28, 2026

### 🚀 New Features

#### Composite Confidence Score (CCS)
- Added `ConfidenceMetrics` system in the backend (`backend/models.py`, `backend/main.py`)
  - `judge_certainty` (human-AI judge self-reported confidence)
  - `inter_model_agreement` (similarity across model answers)
  - `statistical_plausibility` (numerical/magnitude checks)
  - `rule_validation_strength` (compliance with validation rules)
  - `composite_confidence_score` (0.4×judge + 0.3×agreement + 0.2×statistical + 0.1×validation)
- Extended `JudgeResponse` schema to include `confidence_metrics`
- Adjusted judge prompt in `src/hooks/useComparisonState.ts` to request explicit 0-1 confidence from the judge

#### Frontend Confidence UI
- Updated data model in `src/types/index.ts` with `ConfidenceMetrics` and `JudgeResult.confidenceMetrics`
- Updated `src/components/ComparisonResults/JudgeScoreCard.tsx` to show:
  - Numeric CCS badge (High/Medium/Low color-coded)
  - Per-metric bars for judge certainty, agreement, plausibility, and validation strength
  - Responsive and compact presentation
- Added CSS styles in `src/components/ComparisonResults/JudgeScoreCard.css`

### 🔧 Performance & Stability Improvements
- Memory optimization confirmed to be stable across previous enhancements now included in the same release
- Frontend build validated (Vite rollout success)
- Backend syntax checks passed (`python -m py_compile`)

#### Advanced Statistical Analysis for Judge Confidence
- **Z-Score Normalization**: Standardize scores relative to model population means
- **Confidence Intervals**: Probabilistic ranges for score reliability
- **Outlier Detection**: Identify unusually high/low scores that might indicate bias
- **Bayesian Updating**: Incorporate prior knowledge about model performance
- **Credence Calibration**: How well confidence scores match actual accuracy
- **Likelihood Ratios**: Strength of evidence for different quality levels

### 🔧 Technical Improvements

#### Component Architecture Refactoring
- **SideBySideView Enhancement**: Extended to pass `ValidationResult[]` props for badge display
  - Added `getValidationColor()` helper function for score-based coloring
  - Integrated validation badges into `panel-header` alongside winner badges
  - Maintains backward compatibility for comparisons without validation data

#### Type System Updates
- **GeneratedResults Extended**: Added `datasetValidationResult?: ValidationResult` field
- **DatasetValidationRequest**: Exported type for validation API requests
- **ConfidenceMetrics Expansion**: Enhanced to include advanced statistical measures:
  - `agreement_ci_lower/upper`: 95% confidence intervals for inter-model agreement
  - `score_ci_lower/upper`: Confidence intervals for mean score assessments
  - `outlier_ratio`: Proportion of outlier scores detected via Z-score analysis
  - `likelihood_ratio`: Strength of evidence for quality assessment

#### Advanced Statistical Analysis Backend
- **Enhanced Confidence Metrics Calculation** in `backend/main.py`:
  - **Z-Score Normalization**: Detects outliers using Z-score > 2.5 threshold
  - **Confidence Intervals**: Calculates 95% CIs for agreement and score distributions
  - **Outlier Detection**: Identifies anomalous scores that exceed 2.5 standard deviations
  - **Bayesian Updating**: Prior (0.5) + likelihood ratio → posterior odds → calibrated confidence
  - **Credence Calibration**: Adjusts confidence based on score distribution variance (penalty factor: 1/(1+σ/5))
  - **Likelihood Ratios**: Final odds calculation for confidence assessment reliability

### 📊 Data Flow Improvements

#### Validation Pipeline
- **Request Flow**: 
  - Component → `validateDataset()` hook → API POST `/api/validation/dataset`
  - Request includes: name, description, columns array, data_source
  - Response: `ValidationResult` with full issue breakdown
  
- **State Management**:
  - Validation results integrated into `ComparisonResult` type
  - Single dataset results stored in `GeneratedResults`
  - Comparison results keyed by dataset and column name

#### Confidence Metrics Pipeline
- **Calculation Flow**:
  - Judge output parsed for confidence score (0-1)
  - Model metrics collected from judge evaluation
  - `calculate_confidence_metrics()` processes 4 independent metrics:
    1. Judge certainty (direct from judge)
    2. Inter-model agreement (pairwise score similarity)
    3. Statistical plausibility (variance and outlier analysis)
    4. Validation strength (Bayesian-calibrated)
  - Weighted composite: 35% certainty + 25% agreement + 20% plausibility + 20% validation
  
- **Frontend Display**:
  - All 11 metrics rendered with progress bars and numeric values
  - Confidence intervals shown in separate section
  - Color-coded badge indicates overall confidence level
